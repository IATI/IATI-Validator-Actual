Identifiers
ID Severity Description Context (XPath match) Pattern (XPath test)
1.1.1 Warning The activity identifier usually begins with the organisation identifier of the reporting organisation. iati-activity not(starts-with($id, reporting-org/@ref))
1.1.2 Warning There are multiple copies of an activity with this identifier. iati-activity count(//iati-activity[iati-identifier=$id])>1
1.2.1 Warning An organisation identifier should not start or end with spaces or newlines. reporting-org|participating-org|provider-org|receiver-org @ref != functx:trim(@ref)
1.2.2 Improvement No organisation identifier is given. reporting-org|participating-org|provider-org|receiver-org not(@ref)
1.2.3 Improvement An activity identifier is given, but the identifier for the organisation is missing. reporting-org|participating-org|provider-org|receiver-org not(@ref) and (activity-id or receiver-activity-id or provider-activity-id)
1.2.4 Warning The identifier does not conform to the organisation identifier standard. reporting-org|participating-org|provider-org|receiver-org @ref!='' and not( matches(@ref, '^[0-9]{5}$') or matches(@ref, '^[A-Z]{2}(-.+)*$'))
1.2.5 Warning The registration agency part in an organisation identifier should be in uppercase. reporting-org|participating-org|provider-org|receiver-org matches(@ref, '^.+-.+-') and matches($RegistrationAgency, '[a-z]')
1.2.6 Warning The provider-activity-id attribute should not be empty but omitted if you don't have the activity identifier. reporting-org|participating-org|provider-org|receiver-org @provider-activity-id=''
1.2.7 Warning The receiver-activity-id attribute should not be empty but omitted if you don't have the activity identifier. reporting-org|participating-org|provider-org|receiver-org @receiver-activity-id=''

Sector classifications
ID Severity Description Context (XPath match) Pattern (XPath test)
2.1.1 Error One or more sectors in vocabulary X have no percentage: iati-activity count(current-group()[not(@percentage)]) > 0
2.1.2 Error Percentages for sectors in vocabulary X don't add up to 100%. iati-activity sum(current-group()/@percentage) != 100
2.1.3 Improvement You are using sectors in both the activity and transactions in the activity. You should only use them in one place. iati-activity count(sector)>0 and count(transaction/sector)>0

Traceability
ID Severity Description Context (XPath match) Pattern (XPath test)
5.1.1 Improvement The activity is declared to be at hierarchical level X but there is no reference to a parent activity, and no other activity in the dataset refers to this activity as a child activity. iati-activity (@hierarchy > 1) and not(related-activity[@type='1']) and not(//iati-activity/related-activity[@type='2' and @ref=$aid])
100.1.1 Warning Include at least one activity with a transaction of type 11 (incoming commitment) that refers to the Ministry (XM-DAC-7) as the provider, and that refers to a specific activity identifier of the Ministry. iati-activities not(//transaction[transaction-type/@code='11' and provider-org/@ref='XM-DAC-7' and provider-org/@provider-activity-id])
100.2.1 Improvement An activity should contain links to other activities, for instance to indicate where funding comes from or goes to, or how it relates to overarching programmes or underlying projects. This helps understand the complete flow, and prevent double counting. iati-activity not(//@provider-activity-id or //@receiver-activity-id or participating-org/@activity-id or related-activity/@ref)
100.2.2 Warning If the Ministry (XM-DAC-7) is a donor or provider of incoming funds, the activity must have a transaction of type 11 (incoming commitment) iati-activity ( (participating-org[@ref='XM-DAC-7' and @role='1']) or (transaction[provider-org/@ref='XM-DAC-7' and transaction-type/@code='1']) ) and not ( transaction[provider-org/@ref='XM-DAC-7' and transaction-type/@code='11'] )

Financial information
ID Severity Description Context (XPath match) Pattern (XPath test)
7.1.1 Warning The provider-activity-id and receiver-activity-id are the same: financial flows should be between different activities. transaction[provider-org and receiver-org] provider-org/@provider-activity-id eq receiver-org/@receiver-activity-id
7.1.2 Warning The transaction is a commitment from the activity, and has a provider-activity-id that is different from the activity identifier but no receiver-activity-id. transaction[provider-org and receiver-org] upper-case(transaction-type/@code)=('2','C') and provider-org/@provider-activity-id and provider-org/@provider-activity-id != ../iati-identifier and not(receiver-org/@receiver-activity-id)
7.1.3 Warning The transaction is a disbursement from the activity, and has a provider-activity-id that is different from the activity identifier but no receiver-activity-id. transaction[provider-org and receiver-org] upper-case(transaction-type/@code)=('3','D') and provider-org/@provider-activity-id and provider-org/@provider-activity-id != ../iati-identifier and not(receiver-org/@receiver-activity-id)
7.1.4 Warning The transaction is an expenditure from the activity, but has a receiver-activity-id suggesting it may be a disbursement. transaction[provider-org and receiver-org] upper-case(transaction-type/@code)=('4','E') and receiver-org/@receiver-activity-id
7.2.1 Warning The provider and receiver activity identifiers in the transaction should not be the same. transaction[provider-org/@provider-activity-id and receiver-org/@receiver-activity-id] provider-org/@provider-activity-id = receiver-org/@receiver-activity-id
7.3.1 Warning The incoming transaction has no provider organisation identifier or name. transaction[transaction-type/@code=('1','11')] not(provider-org/@ref!='') and not(provider-org/narrative!='')
7.4.1 Warning The outgoing transaction has no receiver organisation identifier or name. transaction[transaction-type/@code=('2','3')] not(receiver-org/@ref!='') and not(receiver-org/narrative!='')
7.5.1 Warning The budget has a value of 0. budget value=0 or value='0'
7.5.2 Error The budget value has no value date. budget not(value/@value-date)
